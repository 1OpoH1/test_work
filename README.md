Платформа для планирования мероприятий с полным набором CRUD операций, геоданными, погодой, импортом/экспортом и периодическими задачами.

## Функционал
### 1. Управление местами проведения (Locations)
- CRUD для мест проведения мероприятий
- Поля: название, гео-координаты (широта/долгота)
- Доступ только для суперпользователя
### 2. Управление мероприятиями (Events)
- CRUD для мероприятий с расширенными полями:
  - Название, описание
  - Множественные изображения
  - Автоматическое создание превью (200px по наименьшей стороне)
  - Даты: публикации, начала, завершения
  - Автор (автоматически привязывается)
  - Место проведения (связь с Location)
  - Погода в месте проведения (автоматическое обновление)
  - Рейтинг (0-25)
  - Статус (черновик/опубликовано/отменено)
### 3. Разграничение прав доступа
- Суперпользователь: полный доступ ко всем операциям
- Обычные пользователи: только просмотр опубликованных мероприятий
- Анонимные пользователи: только просмотр опубликованных мероприятий
### 4. Фильтрация, поиск и пагинация
- Пагинация (10 элементов на странице)
- Сортировка по названию, дате начала, дате завершения
- Фильтрация по:
  - Диапазону дат начала
  - Диапазону дат завершения
  - Месту проведения (множественный выбор)
  - Диапазону рейтинга
- Поиск по названию мероприятия или места проведения
### 5. Импорт и экспорт
- Импорт мероприятий из XLSX-файла (создание мест автоматически)
- Экспорт мероприятий в XLSX с применением фильтров
- Шаблон для импорта
### 6. Периодические задачи (Celery)
- **Публикация мероприятий**: автоматическая публикация при наступлении даты публикации
- **Обновление погоды**: каждые 30 минут для мероприятий на ближайшие 7 дней
- **Email-уведомления**: при публикации мероприятия (настраиваемый список адресатов) (Пока что реализована отправка уведомления в консоль)
### 7. Погода
Реалистичные данные о погоде для каждого мероприятия:
- Температура (°C)
- Влажность (%)
- Атмосферное давление (мм рт. ст.)
- Направление ветра (16 направлений)
- Скорость ветра (м/с)
- Данные автоматически обновляются и нередактируемы
## Технологический стек
- **Backend**: Python 3.12, Django 5.1, Django REST Framework
- **База данных**: PostgreSQL 15
- **Асинхронные задачи**: Celery 5.4, Redis 7
- **Геоданные**: GeoDjango (подготовлено, но используются float поля для простоты)
- **Документация**: drf-spectacular (Swagger/OpenAPI)
- **Тестирование**: pytest, factory-boy
- **Контейнеризация**: Docker, Docker Compose
- **Импорт/экспорт**: openpyxl
- **Обработка изображений**: Pillow
## Установка и запуск
### Предварительные требования
- Python 3.12+
- PostgreSQL 15+
- Redis 7+
- Docker и Docker Compose (для контейнерного запуска)
### Локальный запуск
1. **Клонировать репозиторий**
```bash
git clone <repository-url>
cd event_planner
```
2. **Создать и активировать виртуальное окружение**
    
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows
```


3. **Установить зависимости**
    
```bash
pip install -r requirements.txt
```

4. **Создать файл .env**
    
```env
SECRET_KEY=your-secret-key-here
DEBUG=1
DB_NAME=event_planner
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432
REDIS_HOST=localhost
REDIS_PORT=6379
NOTIFICATION_EMAIL_LIST=admin@example.com,user@example.com
```
5. **Создать базу данных PostgreSQL**
    
```sql
CREATE DATABASE event_planner;
```
6. **Применить миграции**
```bash
python manage.py migrate
```
7. **Создать суперпользователя**
    
```bash

python manage.py createsuperuser
```
8. **Запустить Redis** (отдельно)
    
```bash
redis-server
```
9. **Запустить Celery worker и beat** (в отдельных терминалах)
    

```bash
celery -A event_planner.celery worker --loglevel=info -P eventlet
celery -A event_planner.celery beat --loglevel=info
```

10. **Запустить сервер разработки**
```bash

python manage.py runserver
```

11. **Открыть в браузере**
    

- API: [http://127.0.0.1:8000/api/](http://127.0.0.1:8000/api/)
    
- Swagger: [http://127.0.0.1:8000/api/schema/swagger-ui/](http://127.0.0.1:8000/api/schema/swagger-ui/)
    
- Admin: [http://127.0.0.1:8000/admin](http://127.0.0.1:8000/admin)
    

### Запуск в Docker

1. **Клонировать репозиторий**
    

```bash
git clone https://github.com/1OpoH1/test_work
cd event_planner
```

2. **Создать файл .env** (как указано выше)
    
3. **Запустить контейнеры**
    
```bash
docker-compose up --build
```

4. **Создать суперпользователя** (в отдельном терминале)
```bash
docker-compose exec web python manage.py createsuperuser
```

5. **Применить миграции** (если не применились автоматически)
    
```bash
docker-compose exec web python manage.py migrate
```

6. **Открыть в браузере**
    
- API: [http://localhost:8000/api/](http://localhost:8000/api/)
    
- Swagger: [http://localhost:8000/api/schema/swagger-ui/](http://localhost:8000/api/schema/swagger-ui/)
    
6. **Остановка контейнеров**
    
```bash
docker-compose down
```
# С удалением томов (очистка БД)
```bash
docker-compose down -v
```
